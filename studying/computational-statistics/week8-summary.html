Lecture 8 — The Bootstrap

Course: Computational Statistics (401-3632-00L) Lecture 8 Dr. Jonathan Koh · ETH Zürich · FS2026
Week 8 continues the deep dive into the bootstrap (§6 of the lecture notes), one of the most influential ideas in modern statistics, introduced by Efron in 1979. The key insight is elegantly simple: since we cannot simulate from the unknown true distribution $P$, we instead simulate from an estimated version of it. This week develops the nonparametric and parametric bootstrap in detail, explains how to use them for variance estimation and bias correction, and presents several methods for constructing bootstrap confidence intervals.

1. Recap & Motivation: Why Bootstrap?

In classical statistics, we often rely on analytic formulas to characterise the uncertainty of an estimator $\widehat{\theta}_n = g(Z_1, \ldots, Z_n)$ computed from i.i.d. data $Z_1, \ldots, Z_n \overset{\text{iid}}{\sim} P$. For example, if $\widehat{\theta}_n = \bar{X}_n$, the Central Limit Theorem gives us

$$\sqrt{n}(\bar{X}_n - \mu) \xrightarrow{d} \mathcal{N}(0, \sigma^2),$$

which can be used to build confidence intervals analytically. Similarly, in the Gaussian linear model, exact $t$- and $F$-distributions are available for inference on regression coefficients $\hat{\beta}$.

When classical theory breaks down

In many realistic situations, analytic solutions are unavailable:

  • The exact sampling distribution of $\widehat{\theta}_n$ is unknown or intractable.
  • Asymptotic variances are difficult to derive (e.g. for a sample median, a ratio, or a complex estimator).
  • Asymptotic approximations may be poor for moderate sample sizes $n$.
  • The estimator has no simple closed-form variance (e.g. cross-validated prediction error).

Key insight: if the true distribution $P$ were known, we could just simulate many datasets from $P$, compute $\widehat{\theta}_n$ on each, and approximate the distribution of $\widehat{\theta}_n$ directly by Monte Carlo. Since $P$ is unknown, the bootstrap replaces it with an estimate $\widehat{P}$.

2. The Bootstrap Principle

Definition — Bootstrap Principle (§6.1)

Given observations $Z_1, \ldots, Z_n \overset{\text{iid}}{\sim} P$ (unknown) and an estimator $\widehat{\theta}_n = g(Z_1, \ldots, Z_n)$:

  1. Estimate $P$ by some $\widehat{P}$ (either nonparametrically or parametrically).
  2. Draw $B$ independent bootstrap samples $Z_1^*, \ldots, Z_n^* \overset{\text{iid}}{\sim} \widehat{P}$.
  3. Compute the bootstrap statistic $\widehat{\theta}_n^* = g(Z_1^*, \ldots, Z_n^*)$ for each sample.
  4. Use the empirical distribution of $\widehat{\theta}_n^{*(1)}, \ldots, \widehat{\theta}_n^{*(B)}$ to approximate the sampling distribution of $\widehat{\theta}_n$.

The key substitution is $P \to \widehat{P}$ and $\widehat{\theta}_n \to \widehat{\theta}_n^*$ (real world $\to$ bootstrap world).

2.1 Nonparametric Bootstrap

In the nonparametric bootstrap, we estimate $P$ by the empirical distribution $\widehat{P}_n$, which places equal mass $1/n$ on each observed data point $z_i$. Sampling from $\widehat{P}_n$ is equivalent to sampling with replacement from the original dataset $\{Z_1, \ldots, Z_n\}$.

Definition — Empirical Distribution & Nonparametric Bootstrap

The empirical distribution function is:

$$\widehat{F}_n(z) = \frac{1}{n} \sum_{i=1}^n \mathbf{1}[Z_i \leq z].$$

A bootstrap sample $Z_1^*, \ldots, Z_n^*$ is drawn i.i.d. from $\widehat{F}_n$, i.e. each $Z_i^*$ is chosen uniformly at random (with replacement) from $\{Z_1, \ldots, Z_n\}$.

The resulting bootstrap replicate of the statistic is $\widehat{\theta}_n^* = g(Z_1^*, \ldots, Z_n^*)$.

Intuition: sampling with replacement

When you draw $n$ observations from $n$ with replacement, on average about $1 - 1/e \approx 63.2\%$ of the original data points will appear (some more than once), and about $36.8\%$ will not appear at all. This gives each bootstrap sample a slightly different composition — enough variability to simulate the randomness in $\widehat{\theta}_n$.

2.2 Parametric Bootstrap

If we have good reason to believe the data come from a specific parametric family — say $P = P_\theta$ for some parameter $\theta$ — we can use the parametric bootstrap: estimate $\theta$ to obtain $\widehat{\theta}$, and then sample from the fitted model $P_{\widehat{\theta}}$.

Example — Parametric Bootstrap for Linear Regression

In the linear model $Y = X\beta + \varepsilon$ with $\varepsilon \sim \mathcal{N}(0, \sigma^2 I)$:

  1. Fit the model to get $\widehat{\beta}$ and $\widehat{\sigma}^2$.
  2. For each bootstrap sample $b = 1, \ldots, B$:
    • Draw $\varepsilon_i^* \overset{\text{iid}}{\sim} \mathcal{N}(0, \widehat{\sigma}^2)$.
    • Form $Y_i^* = X_i\widehat{\beta} + \varepsilon_i^*$.
    • Fit the model to $(X, Y^*)$ to obtain $\widehat{\beta}^*$.
  3. Use the distribution of $\widehat{\beta}^*$ to quantify uncertainty in $\widehat{\beta}$.

An alternative is residual resampling: resample from the estimated residuals $\widehat{\varepsilon}_i = Y_i - X_i\widehat{\beta}$ instead of from a fitted Gaussian.

3. The Bootstrap Algorithm

In practice, the distribution of $\widehat{\theta}_n^*$ (conditional on the data) can rarely be computed analytically. Instead, we approximate it by Monte Carlo: draw $B$ bootstrap samples and compute the statistic on each.

Pseudocode — Nonparametric Bootstrap 
Input:  data Z = {Z₁, ..., Zₙ}, statistic g(·), number of replicates B
Output: bootstrap replicates θ̂*₁, ..., θ̂*_B

for b = 1 to B do
    Draw Z*₁, ..., Z*ₙ  i.i.d. from {Z₁, ..., Zₙ}  (with replacement)
    Compute θ̂*_b = g(Z*₁, ..., Z*ₙ)
end for
return θ̂*₁, ..., θ̂*_B

A typical choice is $B = 1{,}000$ to $10{,}000$ replicates. Larger $B$ reduces Monte Carlo error in the approximation.

Example — Bootstrap in R (Linear Regression)

The following code (from the lecture notes, §1.5.2) estimates the standard deviation of regression coefficients via the nonparametric bootstrap:

R 
n <- nrow(data)
B <- 1000
coef_matrix <- matrix(nrow = B, ncol = p)

for (b in 1:B) {
  idx <- sample(1:n, n, replace = TRUE)   # bootstrap sample (indices)
  fit <- lm(Y ~ ., data = data[idx, ])
  coef_matrix[b, ] <- coef(fit)
}

# Bootstrap standard errors
sd_boot <- apply(coef_matrix, 2, sd)

The resulting sd_boot approximates the standard error of each $\widehat{\beta}_j$. For the Gaussian linear model this matches the analytic Std. Error column from summary(lm(...)), but the bootstrap approach is fully general and works even when analytic formulas are unavailable.

4. Bootstrap Variance and Bias Estimation

4.1 Variance Estimation

The bootstrap provides a natural estimator of the variance of $\widehat{\theta}_n$. Recall the true (but unknown) variance is $\text{Var}_P(\widehat{\theta}_n)$. The bootstrap approximates this by the variance under $\widehat{P}$:

$$\widehat{\text{Var}}_{\text{boot}} = \frac{1}{B-1} \sum_{b=1}^{B} \left(\widehat{\theta}_n^{*(b)} - \bar{\theta}^*\right)^2, \quad \bar{\theta}^* = \frac{1}{B}\sum_{b=1}^B \widehat{\theta}_n^{*(b)}.$$

This is simply the sample variance of the bootstrap replicates. Its square root $\widehat{\text{se}}_{\text{boot}} = \sqrt{\widehat{\text{Var}}_{\text{boot}}}$ is the bootstrap standard error.

What the bootstrap variance estimates

The bootstrap estimates $\text{Var}_{\widehat{P}}(\widehat{\theta}_n^*)$, which approximates the true $\text{Var}_P(\widehat{\theta}_n)$. The approximation is good when $\widehat{P}$ is a good estimate of $P$ (i.e. when $n$ is reasonably large). The bootstrap does not give you the exact variance in finite samples.

4.2 Bias Estimation

The bias of $\widehat{\theta}_n$ as an estimator of the true parameter $\theta = \theta(P)$ is defined as:

$$\text{Bias}_P(\widehat{\theta}_n) = \mathbb{E}_P[\widehat{\theta}_n] - \theta(P).$$

In the bootstrap world, the role of $\theta(P)$ is played by $\theta(\widehat{P}) = \widehat{\theta}_n$ (since $\widehat{\theta}_n$ is the "true" parameter in the bootstrap world). The bootstrap estimate of bias is therefore:

$$\widehat{\text{Bias}}_{\text{boot}} = \bar{\theta}^* - \widehat{\theta}_n,$$

where $\bar{\theta}^* = B^{-1}\sum_b \widehat{\theta}_n^{*(b)}$ is the mean of bootstrap replicates and $\widehat{\theta}_n$ is the estimate on the original data.

A bias-corrected estimator can then be formed as:

$$\widehat{\theta}_{\text{bc}} = \widehat{\theta}_n - \widehat{\text{Bias}}_{\text{boot}} = 2\widehat{\theta}_n - \bar{\theta}^*.$$
Bias–Variance trade-off in bias correction

Bias-correcting an estimator typically increases its variance. So unless the bias is large relative to the standard error, it may not be worth correcting. Always consider the mean squared error:

$$\text{MSE}(\widehat{\theta}_n) = \text{Bias}(\widehat{\theta}_n)^2 + \text{Var}(\widehat{\theta}_n).$$

5. Bootstrap Confidence Intervals

Using the bootstrap replicates $\widehat{\theta}_n^{*(1)}, \ldots, \widehat{\theta}_n^{*(B)}$, there are several ways to construct a confidence interval for the true parameter $\theta = \theta(P)$. The main methods are presented below.

5.1 Normal (Wald) Interval

If $\widehat{\theta}_n$ is approximately normally distributed, we can use the bootstrap standard error to form a Wald-type confidence interval:

$$\text{CI}_{\text{normal}} = \left[\widehat{\theta}_n \pm z_{\alpha/2} \cdot \widehat{\text{se}}_{\text{boot}}\right],$$

where $z_{\alpha/2}$ is the relevant standard normal quantile (e.g. $1.96$ for a 95% interval). This is easy to compute, but relies on the normality assumption and ignores any skewness in the bootstrap distribution.

5.2 Percentile Interval

A more direct approach uses the empirical quantiles of the bootstrap distribution. The $(1 - \alpha)$ confidence interval is:

$$\text{CI}_{\text{perc}} = \left[\widehat{\theta}_n^{*(\alpha/2)},\; \widehat{\theta}_n^{*(1-\alpha/2)}\right],$$

where $\widehat{\theta}_n^{*(\alpha/2)}$ is the $\alpha/2$ quantile of the $B$ bootstrap replicates (i.e. the $\lfloor B \cdot \alpha/2 \rfloor$-th sorted value). This is simple and does not require normality, but it can be biased when $\widehat{\theta}_n$ is itself biased.

5.3 Basic (Reverse Percentile) Interval

The basic bootstrap interval is constructed using the bootstrap distribution of the deviation $\widehat{\theta}_n^* - \widehat{\theta}_n$. The idea is to use this bootstrap deviation to approximate the real-world deviation $\widehat{\theta}_n - \theta$. This gives:

$$\text{CI}_{\text{basic}} = \left[2\widehat{\theta}_n - \widehat{\theta}_n^{*(1-\alpha/2)},\; 2\widehat{\theta}_n - \widehat{\theta}_n^{*(\alpha/2)}\right].$$

Note the endpoints are "reflected" around $\widehat{\theta}_n$ — this is why it is also called the reverse percentile interval.

5.4 Studentized (Pivot) Interval

The most theoretically rigorous approach is to bootstrap a studentized statistic (a "pivot"). Define:

$$T_n = \frac{\widehat{\theta}_n - \theta}{\widehat{\text{se}}(\widehat{\theta}_n)}.$$

In the bootstrap world, the analogue is $T_n^* = (\widehat{\theta}_n^* - \widehat{\theta}_n) / \widehat{\text{se}}(\widehat{\theta}_n^*)$, where $\widehat{\text{se}}(\widehat{\theta}_n^*)$ is the standard error of $\widehat{\theta}_n^*$ estimated from a nested inner bootstrap. The interval is then:

$$\text{CI}_{\text{stud}} = \left[\widehat{\theta}_n - T_n^{*(1-\alpha/2)} \cdot \widehat{\text{se}}(\widehat{\theta}_n),\; \widehat{\theta}_n - T_n^{*(\alpha/2)} \cdot \widehat{\text{se}}(\widehat{\theta}_n)\right].$$

This method achieves better accuracy (it is second-order correct) but is computationally expensive because it requires a nested bootstrap loop to estimate $\widehat{\text{se}}(\widehat{\theta}_n^*)$.

6. Nonparametric vs. Parametric Bootstrap

The choice between the two flavours depends on how much you trust a parametric model for the data-generating process:

  • Nonparametric bootstrap: makes no distributional assumptions beyond the data itself. Robust, but may be less efficient when a parametric model is appropriate.
  • Parametric bootstrap: leverages a fitted model $P_{\widehat{\theta}}$. More efficient if the model is correct, but can fail badly if the model is misspecified.
Regression: pairs vs. residuals bootstrap

In the regression context $(X_i, Y_i)$, there are two common nonparametric bootstrap strategies:

  • Pairs bootstrap: resample full pairs $(X_i^*, Y_i^*)$ with replacement. Valid under fixed or random design; robust to heteroskedasticity.
  • Residuals bootstrap: fix the design $X$ and resample residuals $\widehat{\varepsilon}_i^*$ from $\{\widehat{\varepsilon}_1, \ldots, \widehat{\varepsilon}_n\}$. Form $Y_i^* = X_i\widehat{\beta} + \widehat{\varepsilon}_i^*$. Assumes homoskedasticity but is more efficient when the model is correct.

7. Practical Considerations

How large should $B$ be?

The Monte Carlo error in estimating $\text{Var}_{\widehat{P}}(\widehat{\theta}_n^*)$ decreases as $O(1/\sqrt{B})$. In practice:

  • $B = 200$–$500$ is typically sufficient for variance/standard error estimation.
  • $B = 1{,}000$–$2{,}000$ is recommended for percentile confidence intervals.
  • $B = 5{,}000$–$10{,}000$ may be needed for accurate tail quantiles (e.g. 99% intervals).
  • The studentized interval requires an inner bootstrap loop (say $B_\text{inner} = 50$–$100$ per outer replicate), making it computationally demanding: $B \times B_\text{inner}$ total fits.

When does the bootstrap work well?

The bootstrap is consistent — meaning the bootstrap distribution converges to the true sampling distribution — under fairly mild regularity conditions on $g$ and $P$. Key requirements include:

  • The estimator $\widehat{\theta}_n$ should be "smooth" in the data (e.g. differentiable statistics work well; sample maximum or extreme quantiles can be problematic).
  • The sample size $n$ should be large enough that $\widehat{P}_n$ is a good approximation to $P$.
  • The data should be i.i.d. (the standard bootstrap fails for time series or dependent data; modifications such as the block bootstrap are needed).
The bootstrap is not a free lunch

The bootstrap can fail or give poor results for estimators that depend on extreme-order statistics (e.g. the maximum), for very small $n$, or when the data have heavy tails. It approximates $\text{Var}_{\widehat{P}}(\widehat{\theta}_n^*)$, not $\text{Var}_P(\widehat{\theta}_n)$ — the quality of the approximation is only as good as the approximation $\widehat{P} \approx P$.

Key Takeaways

  • Core idea: replace the unknown $P$ with an estimate $\widehat{P}$ and simulate the variability of $\widehat{\theta}_n$ by Monte Carlo in the bootstrap world.
  • Nonparametric bootstrap $=$ sample $n$ observations with replacement from the original data; no distributional assumptions needed.
  • Parametric bootstrap $=$ sample from a fitted model; more efficient when the model is correct, risky if misspecified.
  • Bootstrap variance/bias: compute sample variance/mean of $B$ replicates $\widehat{\theta}_n^{*(b)}$; the difference $\bar{\theta}^* - \widehat{\theta}_n$ estimates the bias.
  • Confidence intervals: four main approaches (Normal, Percentile, Basic, Studentized) — the Studentized interval has the best coverage but requires a nested bootstrap.
  • The bootstrap is one of the most versatile tools in computational statistics, applicable whenever analytic distributions are unavailable or intractable.
