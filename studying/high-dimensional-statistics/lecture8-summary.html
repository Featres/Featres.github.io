Graphical Models and the Road to Uncertainty Quantification

Course: Statistics for High-Dimensional Data Lecture 8 Based on Bühlmann & van de Geer (2011), Ch. 13 & Ch. 10/11

This lecture makes a conceptual shift: instead of predicting a response $Y$ from covariates $X$, we ask how the covariates themselves relate to each other. We develop undirected graphical models, where an edge between two variables encodes a genuine statistical dependence that cannot be explained away by the other variables. We then turn the Lasso loose on this problem (the Graphical Lasso and nodewise regression) and close the lecture by confronting a hard question that will dominate the next chapter: once the Lasso has told us which coefficients are non-zero, how confident are we? Classical p-values and confidence intervals are built on smooth asymptotics, but the Lasso is sparse — a fundamentally non-smooth object. This sets up the de-sparsified (de-biased) Lasso.

1. Recap: High-Dimensional Additive Models

Before diving into graphs, recall where the previous lecture left off. The additive model for a response $Y$ with $p \gg n$ covariates is

$$Y_i = \mu + \sum_{j=1}^{p} f_j^0\!\left(X_i^{(j)}\right) + \varepsilon_i, \qquad i = 1, \ldots, n,$$

where each $f_j^0 : \mathbb{R} \to \mathbb{R}$ is a smooth univariate function with $\mathbb{E}[f_j^0(X_i^{(j)})] = 0$. The goal is a sparse-and-smooth estimator: each $\hat f_j$ is either identically zero or a non-trivial smooth function.

This becomes a Group Lasso-type problem once each $f_j$ is expanded in a basis. The key theoretical takeaway: if only a few $f_j^0$ are non-zero and those that are non-zero are twice continuously differentiable, then

$$\|\hat f - f^0\|_n^2 = O_P\!\left( s_0 \sqrt{\log(p)}\, n^{-4/5} \right),$$

where $s_0$ is the number of non-zero components. We recover the optimal nonparametric rate $n^{-4/5}$ up to a $\sqrt{\log p}$ factor — this $\sqrt{\log p}$ is the price of not knowing which components are active, and it is the hallmark of high-dimensional theory.

Intuition
The model has effectively $O(pn)$ parameters (each smooth function needs $O(n)$ basis coefficients), but $\log(pn)/n = O((\log p + \log n)/n)$ is still small. Sparsity plus smoothness lets us pay only for the few active functions, not all $p$ of them.

We now move to a setting where the covariates themselves are the object of study.

2. Graphical Models: Vocabulary

A graph $G = (V, E)$ is a pair where $V$ is a set of vertices and $E \subseteq V \times V$ is a set of edges. An edge is undirected if $(j,k) \in E$ and $(k,j) \in E$ both hold. Throughout this lecture we consider only undirected graphs.

In a graphical model, vertices correspond to random variables: $V = \{1, \ldots, p\}$ is identified with the components of a random vector

$$X = (X^{(1)}, \ldots, X^{(p)}) \sim P.$$

The pair $(G, P)$ is a graphical model. The edges will encode conditional independence relations among the $X^{(j)}$.

3. Markov Properties and Conditional Independence Graphs

Given a graph, what does a missing edge mean for the distribution? That is the content of a Markov property.

Definition — Pairwise Markov Property
$P$ satisfies the pairwise Markov property with respect to $G$ if, for every pair of unconnected vertices $(j, k) \notin E$ with $j \neq k$,
$$X^{(j)} \perp X^{(k)} \,\big|\, X^{(V \setminus \{j, k\})}.$$
In words: if there is no edge between $j$ and $k$, then $X^{(j)}$ and $X^{(k)}$ are independent once we condition on every other variable.

There is also a stronger, more geometric version. A path is a sequence of vertices $(j_1, \ldots, j_\ell)$ connected by edges in sequence. A set $C$ separates two disjoint sets $A$ and $B$ if every path from $A$ to $B$ passes through some vertex in $C$.

Definition — Global Markov Property
$P$ satisfies the global Markov property with respect to $G$ if for every triple of disjoint sets $A, B, C \subseteq V$ with $C$ separating $A$ and $B$,
$$X^{(A)} \perp X^{(B)} \,\big|\, X^{(C)}.$$

Global $\Rightarrow$ pairwise in general; the converse holds when $P$ has a positive, continuous Lebesgue density (Lauritzen, 1996). The graphical models we work with — where the pairwise property holds — are called Conditional Independence Graphs (CIGs).

Why the conditioning set matters
Pairwise (conditional) independence is not the same as marginal independence. Two variables can be marginally dependent yet conditionally independent given a third — the third is doing all the explaining. A CIG is a map of which dependencies remain after we account for everything else.

4. Gaussian Graphical Models

Now specialise to the Gaussian setting. Assume

$$X = (X^{(1)}, \ldots, X^{(p)}) \sim \mathcal{N}_p(0, \Sigma),$$

with $\Sigma$ positive definite. A Gaussian Graphical Model (GGM) is a CIG whose distribution $P$ is Gaussian. For Gaussians the pairwise and global Markov properties coincide, and — crucially — we can read off edges directly from the matrix $K = \Sigma^{-1}$:

Edge characterisation in a GGM
$$(j, k) \in E \iff X^{(j)} \perp X^{(k)} \,\big|\, X^{(V \setminus \{j, k\})} \;\iff\; \Sigma^{-1}_{j,k} \neq 0.$$

So the edges of a GGM are exactly the non-zero off-diagonal entries of the precision matrix $\Sigma^{-1}$. This is a beautiful reduction: estimating the conditional independence graph becomes estimating the support of a sparse matrix.

4.1 Connection to partial correlations and regression

The precision matrix also encodes partial correlations. Let $C_{j,k} = K_{j,k} / \sqrt{K_{j,j} K_{k,k}}$. Then the partial correlation between $X^{(j)}$ and $X^{(k)}$ given all others is

$$\rho_{jk \mid V \setminus \{j,k\}} = -C_{j,k}.$$

And if we regress $X^{(j)}$ on all other variables,

$$X^{(j)} = \sum_{r \neq j} \beta^{(j)}_r X^{(r)} + \varepsilon^{(j)},$$

then the regression coefficient is $\beta^{(j)}_k = -K_{j,k} / K_{j,j}$. Chain these identities together:

$$(j, k) \in E \;\iff\; \Sigma^{-1}_{j,k} \neq 0 \;\iff\; \rho_{jk \mid V \setminus \{j,k\}} \neq 0 \;\iff\; \beta^{(j)}_k \neq 0 \text{ and } \beta^{(k)}_j \neq 0.$$
The payoff
Estimating a graph edge is the same as doing a variable selection in a regression — and we already have a powerful tool for that: the Lasso. This equivalence is exactly what makes the whole machinery of Chapter 2 portable to graphical modelling.

4.2 The Graphical Lasso (GLasso)

The Gaussian negative log-likelihood (after plugging in $\hat\mu$) is, up to a constant,

$$-n^{-1} \ell(\Sigma^{-1}) = -\log\det(\Sigma^{-1}) + \text{trace}(\hat\Sigma_{\text{MLE}} \Sigma^{-1}),$$

where $\hat\Sigma_{\text{MLE}}$ is the sample covariance. To impose sparsity on $\Sigma^{-1}$, just as we did on $\beta$, we add an $\ell_1$ penalty on the off-diagonal entries:

Definition — Graphical Lasso (GLasso)
$$\hat\Sigma^{-1}(\lambda) = \arg\min_{\Sigma^{-1} \succ 0} \left\{ -\log\det(\Sigma^{-1}) + \text{trace}(\hat\Sigma_{\text{MLE}} \Sigma^{-1}) + \lambda \|\Sigma^{-1}\|_1 \right\},$$
where $\|\Sigma^{-1}\|_1 = \sum_{j < k} |\Sigma^{-1}_{j,k}|$ and the minimisation is over positive-definite matrices. The estimated edge set is
$$\hat E(\lambda) = \{(j, k) : \hat\Sigma^{-1}_{j,k}(\lambda) \neq 0\}.$$

This is a convex problem — efficient algorithms exist (Friedman et al. 2007, Banerjee et al. 2008). Typically, some off-diagonal entries are shrunk exactly to zero, producing a sparse graph. The diagonal is left unpenalised because $1/\Sigma^{-1}_{j,j} = \text{Var}(\varepsilon^{(j)})$, and penalising it would distort the error variances.

4.3 Nodewise Regression (Meinshausen–Bühlmann)

The regression identity at the end of §4.1 suggests a completely different approach: rather than estimating $\Sigma^{-1}$ as a whole, run $p$ separate Lasso regressions, one per node.

Nodewise regression procedure
For each $j = 1, \ldots, p$, regress $X^{(j)}$ on all other variables $\{X^{(r)} : r \neq j\}$ with the Lasso:
$$\hat S^{(j)} = \{r : \hat\beta^{(j)}_r(\lambda) \neq 0\}.$$
Then combine the $p$ estimated neighbourhoods into a graph using one of two rules:
  • OR-rule: edge $(j, k)$ if $k \in \hat S^{(j)}$ or $j \in \hat S^{(k)}$ — more liberal.
  • AND-rule: edge $(j, k)$ if $k \in \hat S^{(j)}$ and $j \in \hat S^{(k)}$ — more conservative.

At the population level (true parameters), the two rules coincide. On samples they disagree, and which one you prefer depends on your tolerance for false positives versus false negatives.

Example — which method when?
Both GLasso and nodewise regression can consistently recover the edge set, but under different conditions. GLasso needs a fairly restrictive irrepresentable condition on $\Sigma^{-1}$ and $\Sigma$ plus large enough non-zero entries. Nodewise regression works under weaker conditions because each regression is a standard Lasso problem.

In practice: GLasso gives you a positive-definite $\hat\Sigma^{-1}$ for free, which is great for prediction and Kullback–Leibler divergence. Nodewise regression is more robust for edge recovery. A common hybrid is to estimate the graph $\hat G$ by nodewise regression and then refit $\hat\Sigma^{-1}$ by MLE under that graph constraint.

5. Ising Model: Binary Variables

Gaussians are not the only game in town. For binary variables $X^{(j)} \in \{0, 1\}$, the analogue is the Ising model:

$$p(x^{(1)}, \ldots, x^{(p)}) = \frac{1}{Z(\gamma)} \exp\!\left( \sum_{j,k} \gamma^{(j)}_k x^{(k)} x^{(j)} \right),$$

with $Z(\gamma)$ a normalising constant. The conditional independence structure mirrors the Gaussian case:

$$X^{(j)} \perp X^{(k)} \,\big|\, X^{(V \setminus \{j,k\})} \iff \gamma^{(j)}_k \neq 0 \text{ and } \gamma^{(k)}_j \neq 0.$$

And there is an analogous regression identity — but this time through logistic regression:

$$\text{logit}\,\mathbb{P}[X^{(j)} = 1 \mid X^{(V \setminus \{j\})}] = \sum_{k \neq j} \beta^{(j)}_k X^{(k)}, \qquad \beta^{(j)}_k = 2\gamma^{(j)}_k.$$

So we can recover the Ising graph by running $p$ nodewise logistic Lasso regressions, using the same OR/AND rules to combine neighbourhoods. The general principle — reduce graph estimation to many regressions — is remarkably robust.

6. Stability Selection with the GLasso

Choosing $\lambda$ for the GLasso is as delicate as for any $\ell_1$ method. Cross-validation gives you the $\lambda$ that predicts well under the negative Gaussian log-likelihood, but that is usually too small — it yields a graph with many spurious edges. Enter stability selection (from Lecture 7).

Example — Riboflavin gene expression
With $p = 160$ gene expressions and $n = 115$ samples, there are $\binom{160}{2} = 12{,}720$ possible edges. The GLasso at its cross-validated $\lambda$ selects far too many. But when combined with stability selection controlling $\mathbb{E}[V] \leq 30$ (expected number of falsely chosen edges), the estimated edge set becomes remarkably stable across $\lambda$ and is much sparser.

As a sanity check: after permuting each variable independently, the true graph is empty. Stability selection correctly returns essentially no edges, whereas plain GLasso still finds hundreds. This is exactly the regime where stability selection shines — it flags unstable edges as artefacts.

7. Uncertainty Quantification: The Problem

So far we have been estimating: point estimates of graphs, point estimates of regression coefficients, variable selection. A Lasso coefficient $\hat\beta_j$ is either zero or non-zero, and that is our answer. But a user reasonably asks: how sure are you?

The classical answer uses p-values and confidence intervals — frequentist uncertainty quantification. The setup is the usual linear model:

$$Y = X\beta^0 + \varepsilon, \qquad p \gg n,$$

and we want to test, for each $j$ (or group $G$),

$$H_{0,j}: \beta^0_j = 0 \quad \text{versus} \quad H_{A,j}: \beta^0_j \neq 0.$$

To get p-values, we would need the asymptotic distribution of the Lasso estimator $\hat\beta(\lambda)$ under $H_0$. That turns out to be very hard.

7.1 Why the Lasso's distribution is pathological

For fixed $p < \infty$ and $n \to \infty$, Knight and Fu (2000) derived the Lasso's asymptotic distribution. It has point mass at zero: with positive probability, the estimator is exactly zero for truly-zero coefficients. This is the sparsity we prized — but it also makes the distribution fundamentally non-smooth and non-Gaussian.

This phenomenon is called super-efficiency (Hodges, 1951): the estimator is more accurate than any unbiased estimator at certain parameter values (here, $\beta^0_j = 0$), but this comes at a cost of irregular behaviour.

Bootstrap fails
The natural fallback — bootstrap or subsampling to approximate the Lasso's distribution — does not work. Formally, if $a_n(\hat\beta_{\text{Lasso},j} - \beta^0_j) \Rightarrow Z$ with $Z$ non-Gaussian, then the bootstrapped version
$$a_n(\hat\beta^*_{\text{Lasso},j} - \hat\beta_{\text{Lasso},j}) \;\not\Rightarrow\; Z \text{ in probability}.$$
The resampled distribution does not converge to the target distribution. Standard bootstrap inference is off the table.

7.2 The cure: desparsify

The fix is to abandon the sparsity of the estimator — at least for inference. We build a de-sparsified (or de-biased) Lasso: a modification that is never exactly zero, but whose distribution is asymptotically Gaussian. We can then use classical Wald-style intervals.

8. The De-Sparsified Lasso

The idea is clever and worth dissecting slowly. It is essentially partialling-out — the classical Frisch–Waugh–Lovell idea — but with a Lasso instead of OLS to handle the $p \gg n$ case.

8.1 Constructing a probe direction

To isolate $\beta^0_j$, we first find a direction that is "as orthogonal as possible" to all other covariates. Run a Lasso of $X^{(j)}$ on $X^{(-j)} = \{X^{(k)} : k \neq j\}$:

$$\hat\gamma^{(j)} = \arg\min_{\gamma} \left\{ \|X^{(j)} - X^{(-j)}\gamma\|_2^2 + \lambda_j \|\gamma\|_1 \right\},$$

and let

$$Z^{(j)} = X^{(j)} - X^{(-j)} \hat\gamma^{(j)}$$

be the residual. Intuitively $Z^{(j)}$ is the part of $X^{(j)}$ that the other covariates cannot predict — a "purified" instrument for the $j$-th direction.

8.2 Projecting $Y$ onto the probe

Compute the inner product $Y^T Z^{(j)} / (X^{(j)})^T Z^{(j)}$. Using $Y = X\beta^0 + \varepsilon$:

$$\frac{Y^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}} = \beta^0_j + \underbrace{\sum_{k \neq j} \frac{(X^{(k)})^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}} \beta^0_k}_{\text{bias}} + \frac{\varepsilon^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}}.$$

The first term is what we want. The last term is noise — mean-zero, Gaussian (under Gaussian $\varepsilon$), tractable. The middle term is the bias: in low dimensions (OLS residual), it would be exactly zero by orthogonality, but with the Lasso residual it does not vanish.

8.3 Subtracting the bias

The bias involves the true unknown $\beta^0_k$. Estimate it with the standard Lasso $\hat\beta$ and subtract:

Definition — De-sparsified Lasso
$$\hat b_j = \frac{Y^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}} - \sum_{k \neq j} \frac{(X^{(k)})^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}} \hat\beta_k, \qquad j = 1, \ldots, p.$$
Equivalently,
$$\hat b_j = \hat\beta_j + \frac{(Y - X\hat\beta)^T Z^{(j)}}{(X^{(j)})^T Z^{(j)}}.$$

The second form is the most enlightening: the de-sparsified Lasso is the standard Lasso, plus a one-step correction using the residuals. The correction undoes the $\ell_1$ shrinkage.

Why this works
The Lasso shrinks coefficients toward zero, introducing bias. The correction step projects the residual $Y - X\hat\beta$ onto the purified direction $Z^{(j)}$, which recovers the missing signal along that direction. The result is not sparse — almost surely $\hat b_j \neq 0$ for every $j$ — but it is asymptotically Gaussian, which is exactly what we need for p-values and confidence intervals.

8.4 Asymptotic distribution

One can show that, under appropriate conditions,

$$\sigma^{-1} \sqrt{n}\; \frac{(X^{(j)})^T Z^{(j)} / n}{\|Z^{(j)}\|_2 / \sqrt{n}} (\hat b_j - \beta^0_j) = W_j + \Delta_j,$$

with $(W_1, \ldots, W_p)^T \sim \mathcal{N}_p(0, \Omega)$ and $\max_j |\Delta_j| = o_P(1)$. The "nice" term $W_j$ is Gaussian; the remainder $\Delta_j$ vanishes uniformly.

This gives the confidence interval:

$$\hat b_j \pm \hat\sigma\, n^{-1/2}\, \frac{\|Z^{(j)}\|_2 / \sqrt{n}}{|(X^{(j)})^T Z^{(j)} / n|}\, \Phi^{-1}(1 - \alpha/2),$$

where $\hat\sigma^2$ can be estimated as $\|Y - X\hat\beta\|_2^2 / n$ or $\|Y - X\hat\beta\|_2^2 / (n - \|\hat\beta\|_0)$. Once you have the machinery, it really is that easy.

8.5 Choosing the tuning parameter $\lambda_j$

For $\hat\beta$ itself we use the usual cross-validated $\hat\lambda_{CV}$. But what about $\lambda_j$ for the nodewise regression of $X^{(j)}$ on $X^{(-j)}$? There is an interesting trade-off:

QuantityScales likeBehaviour as $\lambda_j \downarrow$
Variance of $\hat b_j$$\sigma^2 / \|Z^{(j)}\|_2^2$$\|Z^{(j)}\|_2^2$ shrinks, so variance grows
Bias estimation error$\propto \lambda_j$Shrinks

So smaller $\lambda_j$ inflates the variance but reduces the error from the bias-correction step. The advice is to inflate the variance a bit to keep the bias-correction error under control — this makes type-I error conservative at a small cost in power. A good default choice is not "minimize CV error" but rather something slightly larger that favours bias control.

Three different tools for three adjacent problems, all built from the same $\ell_1$-regularisation idea. The common thread: sparsity is a blessing for estimation and a curse for inference, and clever surgery can reconcile them.

Key Takeaways

  • A graph encodes conditional independence. In a CIG, a missing edge between $j$ and $k$ means $X^{(j)}$ and $X^{(k)}$ are independent given all other variables — dependencies that remain after adjusting for everything else.
  • For Gaussians, edges live in the precision matrix. $(j,k) \in E$ if and only if $\Sigma^{-1}_{j,k} \neq 0$. This reduces graph estimation to estimating the support of a sparse matrix, or equivalently (via regression identities), to many variable-selection problems.
  • Two Lasso-based recipes for Gaussian graphs. GLasso directly $\ell_1$-penalises the Gaussian likelihood over $\Sigma^{-1}$ and gives a positive-definite estimator. Nodewise regression runs $p$ separate Lassos and combines neighbourhoods with AND/OR rules — more robust for edge recovery under weaker conditions. Stability selection tames the tuning-parameter problem.
  • Sparsity breaks classical inference. The Lasso's asymptotic distribution has a point mass at zero ("super-efficiency"), so bootstrap and subsampling are inconsistent. We cannot naively attach p-values or confidence intervals to Lasso coefficients.
  • The de-sparsified Lasso rescues inference. Starting from the Lasso $\hat\beta$, add a one-step correction using residuals projected onto a Lasso-purified direction. The result $\hat b_j$ is never exactly zero but is asymptotically Gaussian, enabling Wald-style confidence intervals $\hat b_j \pm \hat\sigma_{\hat b_j} z_{1 - \alpha/2}$.
  • Tuning involves a bias–variance trade-off. The nodewise regression tuning parameter $\lambda_j$ controls the balance between variance (smaller $\lambda_j$ ⟹ larger variance) and bias-correction error (smaller $\lambda_j$ ⟹ smaller error). A slightly over-regularised $\lambda_j$ trades a bit of power for robust type-I error control.
