01 // summary

An exploratory tool for understanding how ordinary least squares regression reshapes itself as new observations arrive. The user adds points to a canvas; the estimator recomputes β̂ = (XᵀX)⁻¹Xᵀy on every tick and redraws the line.

Built as a static, dependency-free page so it runs anywhere a browser does — no server, no build step, no heavy framework.

Motivation

Textbook OLS is usually presented as a closed-form solution and a static residual plot. This page makes the estimator a live object: every click adds an observation, recomputes the coefficients, and redraws. The hope is that the mechanics become intuitive — leverage, influence, sensitivity to outliers — rather than abstract.

§

How it works

  • Observations live in a small matrix X ∈ ℝⁿˣ² (intercept + x) and vector y ∈ ℝⁿ.
  • On every insertion, β̂ is recomputed by solving XᵀXβ̂ = Xᵀy with a 2×2 inverse.
  • The canvas redraws the fit line and the residual segments for every point.
javascript
// Closed-form OLS for a simple 1D regression.
function fit(points) {
  const n = points.length;
  if (n < 2) return null;
  let sx = 0, sy = 0, sxx = 0, sxy = 0;
  for (const [x, y] of points) {
    sx += x; sy += y; sxx += x*x; sxy += x*y;
  }
  const mx = sx / n, my = sy / n;
  const slope     = (sxy - n*mx*my) / (sxx - n*mx*mx);
  const intercept = my - slope * mx;
  return { slope, intercept };
}

What I learned

  • How quickly OLS breaks under leverage — one point far from the mean of x can tilt the line dramatically.
  • That the same estimator, rendered interactively, makes asymptotic arguments feel local and concrete.
  • A small amount of animation (≈200 ms ease on the line) makes each update legible.
