Unsupervised Learning: Dimension Reduction

Course: Introduction to Machine Learning Lecture 16 Spring 2026, Prof. Andreas Krause

This lecture introduces dimensionality reduction, the unsupervised counterpart of regression: given high-dimensional data, we want to find low-dimensional representations that preserve as much structure as possible. We focus on three model-based approaches — Principal Component Analysis (PCA) as the linear workhorse, Kernel PCA as its nonlinear extension via the kernel trick, and neural network autoencoders as a flexible parametric nonlinear alternative. The three methods all minimize a reconstruction error, which ties them to a common theme: supervising the model with the data itself.

1. Dimensionality Reduction: Motivation

Given a dataset \(\mathcal{D} = \{x_1, \dots, x_n\} \subseteq \mathbb{R}^d\), the goal is to find low-dimensional embeddings \(z_1, \dots, z_n \in \mathbb{R}^k\) with \(k \ll d\). A few reasons why this is useful:

  • Compression. Save memory and speed up downstream computations.
  • Visualization. With \(k \in \{1, 2, 3\}\), we can plot data and gain intuition through exploratory analysis.
  • Feature learning. Cleaner representations can improve generalization of downstream classifiers and regressors by stripping away noise.
  • Structure discovery. Embeddings often reveal meaningful latent factors (think: "baldness" or "gender" directions in a face-image latent space).
Manifold hypothesis
High-dimensional real-world data (images, text, audio) typically lie on or near a much lower-dimensional manifold. Images of faces, for instance, are constrained by symmetries and correlations that drastically reduce the effective degrees of freedom. Dimensionality reduction exploits this structure.

1.1 Linear vs. nonlinear methods

We focus on model-based approaches: we fit a mapping \(f : \mathbb{R}^d \to \mathbb{R}^k\) to the training data and reuse it on new points.

  • Linear: \(f(x) = W x\) for some matrix \(W\). Example: PCA.
  • Nonlinear (parametric or non-parametric): \(f\) is a general nonlinear function. Examples: Kernel PCA, neural-network autoencoders.

The central design question is: which mappings should we prefer? PCA picks the linear mapping that reconstructs the data best in the Euclidean sense.

2. Principal Component Analysis (PCA)

PCA is the canonical linear dimension-reduction technique. It answers the question: what is the best \(k\)-dimensional subspace of \(\mathbb{R}^d\) on which to project the data, where "best" means minimum reconstruction error?

2.1 The one-dimensional case (\(k=1\))

Assume the data is centered, i.e. \(\frac{1}{n}\sum_i x_i = 0\) (otherwise we subtract the mean first). For \(k=1\) we want to approximate each \(x_i \in \mathbb{R}^d\) as a scalar \(z_i \in \mathbb{R}\) times a direction \(w \in \mathbb{R}^d\):

\[ x_i \approx z_i w. \]

To fix the scale ambiguity (the pairs \((z, w)\) and \((\alpha z, \alpha^{-1} w)\) give identical reconstructions), we require \(\|w\|_2 = 1\). PCA then jointly solves:

\[ w^*, z_1^*, \dots, z_n^* \;=\; \arg\min_{\|w\|_2 = 1,\; z_i \in \mathbb{R}}\; \sum_{i=1}^n \|x_i - z_i w\|_2^2. \]

For a fixed \(w\), the optimal scalar is the orthogonal projection \(z_i^* = w^\top x_i\). Substituting and simplifying, the problem becomes:

\[ w^* = \arg\max_{\|w\|_2 = 1}\; \sum_{i=1}^n (w^\top x_i)^2 = \arg\max_{\|w\|_2 = 1}\; w^\top \Sigma w, \]

where \(\Sigma = \frac{1}{n}\sum_{i=1}^n x_i x_i^\top = \frac{1}{n} X^\top X\) is the empirical covariance matrix. So minimizing reconstruction error is the same as maximizing the variance of the projected data — two perspectives, one solution.

Lemma 12.1 — First principal component
For the problem \(w^* = \arg\max_{\|w\|_2=1} w^\top \Sigma w\), the optimum is \(w^* = v_1\), the eigenvector of \(\Sigma\) associated with its largest eigenvalue \(\lambda_1\).
Sketch of the proof
Since \(\Sigma\) is symmetric PSD, its eigenvectors \(v_1, \dots, v_d\) form an orthonormal basis. Write \(w = \sum_i \alpha_i v_i\). Then \(w^\top \Sigma w = \sum_i \alpha_i^2 \lambda_i\) and \(\|w\|_2^2 = \sum_i \alpha_i^2 = 1\). This means we are distributing "probability mass" \(\alpha_i^2\) over the \(\lambda_i\)'s; the maximum is achieved by putting all mass on the largest eigenvalue, giving \(w^* = \pm v_1\).

2.2 The general case (\(k \geq 1\))

For \(k > 1\), we want a matrix \(W \in \mathbb{R}^{d \times k}\) with orthonormal columns (i.e. \(W^\top W = I_k\)) and encodings \(z_i \in \mathbb{R}^k\) solving:

\[ W^*, z_1^*, \dots, z_n^* \;=\; \arg\min_{W^\top W = I_k,\; z_i \in \mathbb{R}^k}\; \sum_{i=1}^n \|W z_i - x_i\|_2^2. \]
Theorem 12.2 — PCA solution
Let \(\Sigma = \sum_{i=1}^d \lambda_i v_i v_i^\top\) be the eigen-decomposition of the empirical covariance with \(\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d\). Then the optimal solution is \[ W^* = (v_1 \mid v_2 \mid \cdots \mid v_k), \qquad z_i^* = W^{*\top} x_i. \] In words: the top \(k\) principal components are the top \(k\) eigenvectors of \(\Sigma\), and the encoding is just the orthogonal projection onto their span.

The variance explained along direction \(v_j\) is exactly the eigenvalue:

\[ \frac{1}{n}\sum_{i=1}^n (v_j^\top x_i)^2 = v_j^\top \Sigma v_j = \lambda_j. \]

PCA acts as an orthogonal projection \(x \mapsto W^* W^{*\top} x\) onto the \(k\)-dimensional subspace spanned by the top eigenvectors. This operator is idempotent (\(P^2 = P\)), as any projection must be.

PCA is not regression
The PC1 line differs from a regression line of \(x_2\) on \(x_1\). Regression minimizes vertical distances (in the direction of the response); PCA minimizes the perpendicular distance to the principal-component line. Both look similar at a glance but answer different questions.

3. PCA via Singular Value Decomposition

In practice, PCA is almost never computed by forming \(\Sigma\) and diagonalizing it. Instead, we use the Singular Value Decomposition (SVD) of the data matrix \(X \in \mathbb{R}^{n \times d}\):

\[ X = U S V^\top, \]

where \(U \in \mathbb{R}^{n \times n}\) and \(V \in \mathbb{R}^{d \times d}\) are orthogonal, and \(S \in \mathbb{R}^{n \times d}\) is diagonal with entries (the singular values) in decreasing order. Plugging this into the covariance:

\[ n \Sigma = X^\top X = V S^\top U^\top U S V^\top = V S^\top S V^\top = V D V^\top, \]

where \(D = S^\top S\) is diagonal. This is exactly the eigen-decomposition of \(n \Sigma\)! Therefore:

PCA ↔ SVD connection
The top \(k\) principal components of \(X\) are the first \(k\) columns of \(V\). Eigenvalues of \(\Sigma\) satisfy \(\lambda_j = \sigma_j^2 / n\) where \(\sigma_j\) are the singular values.

SVD-based implementations (as in scikit-learn) are numerically more stable and often faster than forming \(\Sigma\) explicitly. 

from sklearn.decomposition import PCA
pca = PCA(n_components=2)
pca.fit(X)
X_transformed = pca.transform(X)

Bonus: Eckart–Young

Theorem 12.3 — Eckart–Young
Let \(X\) have rank \(r\) and define the truncated SVD \(\tilde X_k = \sum_{j=1}^k \sigma_j u_j v_j^\top\). Then for any \(k \leq r\), \[ \tilde X_k \in \arg\min_{\mathrm{rank}(Y) \leq k} \|X - Y\|_F^2. \] Hence PCA provides the best rank-\(k\) approximation to \(X\) in the Frobenius norm, and the minimum reconstruction error equals \(\sum_{j=k+1}^d \lambda_j\) — the variance of the discarded components.

4. Choosing the Number of Components \(k\)

How many components should we keep? It depends on what we want to do with them.

Use case Strategy for \(k\)
Visualization \(k \in \{1, 2, 3\}\) by inspection — you need something you can plot.
Feature induction for a downstream model Cross-validation on the downstream task.
Compression Pick \(k\) so a fixed fraction (e.g. 95%) of total variance is retained.
Exploratory Scree plot & look for the "elbow" where the curve flattens.

The scree plot shows explained variance against the number of components. The marginal gain from adding components decreases (diminishing returns), and the "elbow" is a common heuristic choice — analogous to the same trick used for choosing \(k\) in \(k\)-means clustering.

5. PCA and k-Means: A Unified View

At first glance PCA and \(k\)-means look unrelated. In fact they solve the same optimization problem with different constraints. Compare:

PCA problem
\[ (W, z_1, \dots, z_n) = \arg\min_{W^\top W = I_k,\; z \in \mathbb{R}^k} \sum_{i=1}^n \|W z_i - x_i\|_2^2. \]
k-Means problem (equivalent form)
\[ (W, z_1, \dots, z_n) = \arg\min_{W \text{ arbitrary},\; z_i \in E_k} \sum_{i=1}^n \|W z_i - x_i\|_2^2, \] where \(E_k = \{e_1, \dots, e_k\}\) are the standard unit vectors. Here the columns of \(W\) are the cluster centroids \(\mu_1, \dots, \mu_k\), and \(z_i\) encodes the hard cluster assignment.

Both objectives are \(\sum_i \|W z_i - x_i\|_2^2\) — the same reconstruction loss. The only difference is the constraint set for \(W\) and \(z_i\). This insight generalizes to the broader family of matrix factorization methods, which aim to write \(X \approx W Z\) with various constraints on \(W\) and \(Z\) (explored in depth in the Computational Intelligence Lab).

6. Kernel PCA

PCA can only find linear subspaces. But data often lies on a curved manifold where no linear projection captures the structure well (think of points on a circle, or a swiss-roll surface). Kernel PCA solves this with the same trick we used for ridge regression and SVMs: run PCA in a high-dimensional feature space implicitly defined by a kernel.

6.1 The idea

Let \(\phi : \mathbb{R}^d \to \mathbb{R}^p\) be a (possibly very high- or infinite-dimensional) feature map, and let \(k(x, x') = \langle \phi(x), \phi(x') \rangle\) be the associated kernel. We would like to run PCA on the featurized points \(\phi(x_1), \dots, \phi(x_n)\) — but \(p\) may be enormous or infinite, so we cannot form them explicitly.

Recap of kernels
A function \(k : \mathcal{X} \times \mathcal{X} \to \mathbb{R}\) is a valid kernel iff it is symmetric and positive semi-definite. By the representer-style argument we saw earlier, computations that depend only on inner products \(\langle \phi(x), \phi(x') \rangle\) can be carried out using \(k(x, x')\) directly. Popular choices include polynomial kernels and the Gaussian RBF kernel, which corresponds to an infinite-dimensional feature map.

6.2 Recipe

Since PCA only uses inner products of the data (through \(\Sigma = \frac{1}{n} X^\top X\) and its eigenvectors), we can recast everything using the kernel matrix \(K \in \mathbb{R}^{n \times n}\) with \(K_{ij} = k(x_i, x_j)\). The principal components in feature space can be expressed as linear combinations of the featurized data points, and the corresponding coefficients are obtained from the eigenvectors of a centered version of \(K\). The result is a nonlinear embedding \(z : \mathbb{R}^d \to \mathbb{R}^k\) that can capture curved structure — at the price of \(\mathcal{O}(n^2)\) memory for storing \(K\).

7. Neural Network Autoencoders

Autoencoders take yet a third route: parameterize the nonlinear encoder and decoder directly with neural networks, and train them with gradient descent on the same reconstruction objective.

7.1 Architecture

An autoencoder is a neural network \(f : \mathbb{R}^d \to \mathbb{R}^d\) of the form

\[ f(x) = g_{\text{dec}}\big(g_{\text{enc}}(x)\big), \]

where

  • the encoder \(g_{\text{enc}} : \mathbb{R}^d \to \mathbb{R}^k\) squeezes the input through a narrow hidden layer — the bottleneck — of dimension \(k \ll d\), producing the latent code \(z = g_{\text{enc}}(x)\);
  • the decoder \(g_{\text{dec}} : \mathbb{R}^k \to \mathbb{R}^d\) maps \(z\) back to the input space, producing the reconstruction \(\hat{x} = g_{\text{dec}}(z)\).

Training minimizes the reconstruction loss over the parameters of both networks:

\[ \min_{\theta_{\text{enc}},\, \theta_{\text{dec}}}\; \sum_{i=1}^n \|x_i - g_{\text{dec}}(g_{\text{enc}}(x_i))\|_2^2. \]

7.2 The connection to PCA

Linear autoencoders recover PCA
If both the encoder and decoder are linear (i.e. identity activation), then the reconstruction loss is exactly the PCA objective \(\sum_i \|W_{\text{dec}} W_{\text{enc}} x_i - x_i\|_2^2\). The optimal subspace spanned by the learned weights is the PCA subspace — the weights themselves may differ from the PCA eigenvectors (the solution is not unique), but the span matches. In this sense, PCA is a linear autoencoder.

With nonlinear activations (e.g. ReLU, tanh) the autoencoder can learn a genuinely nonlinear manifold — something PCA cannot do. This is the main advantage.

7.3 Use in practice

Autoencoders power a wide range of applications: denoising, anomaly detection, pretraining, and — most notably — variational autoencoders (VAEs), whose latent spaces are still used as the representation space of modern generative models like Stable Diffusion. Varying a single coordinate \(z_j\) in a well-trained VAE often corresponds to a semantically meaningful direction (e.g. "smile", "hair colour"), illustrating how the learned features can align with human-interpretable factors.

8. Comparison of the Three Methods

Method Mapping Training Handles nonlinearity? Scalability
PCA Linear projection \(z = W^\top x\) Closed form via SVD/eigendecomposition No \(\mathcal{O}(\min(n d^2, n^2 d))\), typically cheap
Kernel PCA Nonlinear via implicit feature map \(\phi\) Eigendecomposition of centered \(K \in \mathbb{R}^{n \times n}\) Yes (kernel-defined) \(\mathcal{O}(n^2)\) memory — limits \(n\)
Autoencoder Nonlinear, learned by SGD Gradient descent; local optima possible Yes (very flexible) Scales to very large \(n\); requires more tuning

Key Takeaways

  • Dimension reduction finds embeddings that preserve structure. Useful for compression, visualization, feature learning and interpretability, and motivated by the manifold hypothesis: real-world data lives on low-dimensional manifolds inside high-dimensional spaces.
  • PCA minimizes reconstruction error equivalently to maximizing projected variance. The solution is the top \(k\) eigenvectors of the empirical covariance matrix \(\Sigma\), giving a linear projection with a closed-form optimum.
  • Use SVD in practice. The right singular vectors of the data matrix \(X\) are precisely the principal components; SVD avoids forming \(\Sigma\) and is numerically stable. By Eckart–Young, truncated SVD is the best rank-\(k\) approximation of \(X\) in Frobenius norm.
  • PCA and k-means share a common structure. Both minimize \(\sum_i \|W z_i - x_i\|_2^2\); only the constraints on \(W\) and \(z_i\) differ. This is the gateway to the matrix-factorization family.
  • Kernel PCA lifts PCA to nonlinear manifolds. By running PCA implicitly in the feature space of a kernel, we capture curved structure without ever forming the feature vectors — trading computational scalability for expressiveness.
  • Autoencoders generalize PCA with neural networks. A linear autoencoder recovers the PCA subspace; nonlinear activations give the flexibility to learn arbitrary manifolds. They are the foundation for modern representation learning and generative models like VAEs.
