Unsupervised Learning: Clustering

Course: Introduction to Machine Learning Lecture 15 Spring 2026, Prof. Andreas Krause

This lecture opens the unsupervised learning part of the course, where we drop target labels and instead look for structure inside raw data. The central question is clustering: given a set of points, how can we group them so that similar points end up together and dissimilar ones stay apart? We focus on the k-means algorithm — its problem formulation, Lloyd's heuristic, convergence guarantees, smart initialization via k-means++, and finally the hard question of how to pick the number of clusters $k$.

1. From Supervised to Unsupervised Learning

So far the course has focused on supervised learning: we were given a dataset $\mathcal{D} = \{(x_i, y_i)\}_{i=1}^n$ and learned a map from inputs $x_i$ to targets $y_i$. In unsupervised learning we drop the $y_i$'s entirely and work with

$$\mathcal{D} = \{x_i \in \mathbb{R}^d : i = 1, \dots, n\}.$$

The goal is to discover structure that was never explicitly labelled. Typical tasks include clustering (unsupervised classification), dimensionality reduction (unsupervised regression), and generative modeling / density estimation.

Why bother without labels?
Labels are expensive. In many real datasets — user logs, genomic measurements, raw text scraped from the web — we have billions of observations but zero annotations. Unsupervised methods let us compress data, discover latent groups, find anomalies, learn useful features for downstream supervised tasks, and explore a dataset before committing to a model.
Supervised taskUnsupervised analog
Classification (discrete $y$)Clustering
Regression (continuous $y$)Dimensionality reduction
Conditional $p(y \mid x)$Density estimation / generative modeling

2. What is Clustering?

Clustering (informal)
Given data points $x_1, \dots, x_n$, partition them into groups (clusters) such that
  • similar points lie in the same cluster, and
  • dissimilar points lie in different clusters.
Points are typically embedded in a Euclidean space $\mathbb{R}^d$, with the $\ell_2$ distance $\|x_i - x_j\|_2$ measuring dissimilarity. Other metrics or similarity functions work too.

A closely related task is anomaly / outlier detection: identifying points that "don't fit well" into any cluster.

2.1 Three broad families of clustering methods

FamilyIdeaExamples
Hierarchical Build a tree (bottom-up or top-down) recording distances among points. Single-linkage, average-linkage clustering
Partitioning Define a cost over partitions of a graph and optimise it. Spectral clustering, graph-cut methods
Model-based Maintain explicit cluster "models" and assign each point to the one it fits best. k-means, Gaussian Mixture Models (GMMs)
Why model-based?
Hierarchical and partitioning methods are designed for a fixed dataset — they give you a clustering of the points you had, but no natural rule for assigning new points. Model-based methods learn an object (e.g. centroids) that generalises to unseen data. This is why the lecture concentrates on k-means.

3. The k-Means Problem

In k-means, each cluster is represented by a single point — its center or centroid $\mu_j \in \mathbb{R}^d$. Every data point is assigned to the closest center, and we denote the assignment of $x_i$ by $z_i \in \{1, \dots, k\}$:

$$z_i = \arg\min_{j \in \{1, \dots, k\}} \|x_i - \mu_j\|_2.$$

Since this rule applies to any $x \in \mathbb{R}^d$, not only the training points, the centers induce a partition of the whole space known as a Voronoi partition. That's what lets us generalise the clustering model to new points.

Definition: The k-means objective
Given centers $\mu = (\mu_1, \dots, \mu_k)$, the k-means cost (empirical risk) is $$\hat{R}(\mu) = \sum_{i=1}^{n} \min_{j \in \{1, \dots, k\}} \|x_i - \mu_j\|_2^2 = \sum_{i=1}^{n} \|x_i - \mu_{z_i}\|_2^2.$$ The k-means problem is $$\mu^\star \in \arg\min_{\mu_1, \dots, \mu_k \in \mathbb{R}^d} \hat{R}(\mu_1, \dots, \mu_k).$$
Warning: Non-convex & NP-hard
Although the squared distance $\|x_i - \mu_j\|_2^2$ is convex in $\mu_j$ individually, the objective $\hat{R}(\mu)$ is non-convex because of the outer $\min_j$ (which couples the assignments to the centers). In fact the problem is NP-hard in general, so we cannot expect to solve it optimally. Practical implementations therefore rely on heuristics — most famously Lloyd's.

4. Lloyd's Heuristic

Lloyd's heuristic is the standard iterative algorithm for k-means. It alternates between two cheap steps: freeze the centers and update the assignments, then freeze the assignments and update the centers.

Algorithm: Lloyd's heuristic
  1. Initialize cluster centers $\mu^{(0)} = (\mu_1^{(0)}, \dots, \mu_k^{(0)})$.
  2. For $t = 1, 2, \dots$ until convergence:
    1. Assignment step. Assign each point to its closest center: $$z_i^{(t)} \leftarrow \arg\min_{j \in \{1, \dots, k\}} \|x_i - \mu_j^{(t-1)}\|_2, \quad i = 1, \dots, n.$$
    2. Update step. Recompute each center as the mean of the points assigned to it: $$\mu_j^{(t)} \leftarrow \frac{1}{n_j^{(t)}} \sum_{i : z_i^{(t)} = j} x_i, \quad j = 1, \dots, k,$$ where $n_j^{(t)} = |\{i : z_i^{(t)} = j\}|$.

Each iteration costs $\mathcal{O}(n k d)$: for each of the $n$ points you compute $k$ distances in $\mathbb{R}^d$.

4.1 Why does step 2 use the mean?

The update rule isn't arbitrary — the mean is the optimal center for a fixed assignment. This is a basic fact worth stating explicitly.

Lemma 11.1 — Center of mass minimises squared distances
Given points $x_1, \dots, x_n \in \mathbb{R}^d$, the function $$L(\mu) = \frac{1}{n} \sum_{i=1}^n \|x_i - \mu\|_2^2$$ is minimized by the center of mass $\mu^\star = \frac{1}{n} \sum_{i=1}^n x_i$.

Proof sketch

Add and subtract $\mu^\star$ and expand the square:

$$ \begin{aligned} L(\mu) &= \frac{1}{n}\sum_{i=1}^n \|(x_i - \mu^\star) + (\mu^\star - \mu)\|_2^2 \\ &= \frac{1}{n}\sum_{i=1}^n \|x_i - \mu^\star\|_2^2 + \frac{2}{n}\sum_{i=1}^n (x_i - \mu^\star)^\top (\mu^\star - \mu) + \|\mu^\star - \mu\|_2^2. \end{aligned} $$

The cross term vanishes because $\sum_i (x_i - \mu^\star) = 0$ by definition of the mean. What remains is

$$L(\mu) = \underbrace{\frac{1}{n}\sum_{i=1}^n \|x_i - \mu^\star\|_2^2}_{= L(\mu^\star)} + \|\mu^\star - \mu\|_2^2 \geq L(\mu^\star),$$

with equality iff $\mu = \mu^\star$. $\blacksquare$

Intuition
Both steps of Lloyd are greedy local moves on the joint cost $\hat{R}(\mu, z) = \sum_i \|x_i - \mu_{z_i}\|_2^2$: the assignment step optimises $\hat{R}$ over $z$ with $\mu$ fixed, and the update step optimises $\hat{R}$ over $\mu$ with $z$ fixed. This is a form of coordinate descent.

5. Convergence and Limitations

5.1 Monotone decrease of the cost

Theorem 11.2 — Lloyd's heuristic monotonically decreases the cost
Let $z^{(t)}, \mu^{(t)}$ denote the assignments and centers at iteration $t$. Define $$\hat{R}(\mu, z) = \sum_{i=1}^n \|x_i - \mu_{z_i}\|_2^2.$$ Then for every $t \geq 1$, $$\hat{R}(\mu^{(t)}, z^{(t)}) \geq \hat{R}(\mu^{(t+1)}, z^{(t+1)}).$$

Proof idea (two inequalities, one per step)

The assignment step gives

$$\hat{R}(\mu^{(t)}, z^{(t)}) \geq \min_{z} \hat{R}(\mu^{(t)}, z) = \hat{R}(\mu^{(t)}, z^{(t+1)}),$$

because by construction each $z_i^{(t+1)}$ is chosen to minimise $\|x_i - \mu_j^{(t)}\|_2$. The update step gives

$$\hat{R}(\mu^{(t)}, z^{(t+1)}) \geq \min_{\mu} \hat{R}(\mu, z^{(t+1)}) = \hat{R}(\mu^{(t+1)}, z^{(t+1)}),$$

because Lemma 11.1 tells us that the cluster-wise mean is the optimal center for any fixed assignment. Chaining the two inequalities finishes the proof. $\blacksquare$

Because the cost is a non-negative, monotonically decreasing sequence, the algorithm must converge. But it converges only to a local optimum — and possibly a bad one.

5.2 What can go wrong

Challenges with k-means
  • Local optima only. The algorithm converges but not necessarily to the global minimum. Performance strongly depends on initialization.
  • Worst-case runtime. The number of iterations required can be exponential in the worst case, even in the plane. In practice, however, convergence is usually fast.
  • Wrong inductive bias. The $\ell_2$ cost imposes spherical, roughly equally-sized clusters. Elongated, curved, or density-varying clusters are modelled badly.
  • Choosing $k$ is hard. There is no $k$ that is "objectively" right.
Kernel k-means can partially fix the shape issue; for $k$ we need heuristics (next section).

6. Initialization: k-means++

Since Lloyd's heuristic only finds a local optimum, smart initialization matters a lot. Three common schemes:

  1. Multiple random restarts. Run Lloyd with many random initializations and keep the lowest-cost result. Simple, but expensive.
  2. Farthest points heuristic. Pick the first center uniformly at random, then repeatedly pick the point farthest from the currently chosen centers. Often works well but is very sensitive to outliers (an isolated outlier is almost guaranteed to become a center).
  3. k-means++ seeding. A randomised version of the farthest-points idea that is robust to outliers.
Algorithm: k-means++ initialization
  1. Sample the first center uniformly among the data points: $$\mu_1^{(0)} = x_i, \quad i \sim \operatorname{Uniform}(\{1, \dots, n\}).$$
  2. For $j = 2, \dots, k$, draw the $j$-th center $\mu_j^{(0)} = x_i$ with probability $$p(i) \propto \min_{m \in \{1, \dots, j-1\}} \|x_i - \mu_m^{(0)}\|_2^2.$$
  3. Then run Lloyd's heuristic starting from these centers.
Why is k-means++ clever?
The probability is proportional to squared distance, not maximised over it. Far-away points are more likely to be picked — so cluster centers spread out — but a lone outlier no longer gets chosen with probability one. Randomness + the squared distance weighting = robust, balanced coverage.

Theoretical guarantee

One of the appealing features of k-means++ is that, unlike plain Lloyd, it comes with a provable approximation guarantee:

$$\mathbb{E}\big[\hat{R}(\mu^{\text{k-means++}})\big] \leq \mathcal{O}(\log k) \cdot \min_{\mu} \hat{R}(\mu).$$

In words: the expected cost after k-means++ seeding is within a logarithmic factor in $k$ of the true optimum. This is why scikit-learn and most other libraries use it as the default init.

from sklearn.cluster import KMeans

# k-means++ is the default initialization scheme
kmeans = KMeans(n_clusters=3, init="k-means++", n_init=10)
kmeans.fit(X_train)
predictions = kmeans.predict(X_test)

7. Model Selection: Choosing k

Choosing $k$ is the clustering analog of choosing model complexity in supervised learning — and it's much harder here.

Why cross-validation fails
Increasing $k$ always decreases $\hat{R}$. Even on a held-out validation set, adding more centers gives strictly lower cost (in the limit $k = n$, every point has its own center and $\hat{R} = 0$). So there is no U-shape to cross-validate against. Traditional model selection by validation loss simply does not work.

7.1 The elbow heuristic

Plot $\hat{R}$ against $k$. The curve is typically concave-decreasing with diminishing returns: sharp improvement for small $k$, then it flattens. Choose $k$ at the elbow — the kink beyond which additional clusters barely help.

7.2 Regularization

A more principled-looking alternative is to add a complexity penalty proportional to $k$:

$$\hat{R}_\lambda(\mu_1, \dots, \mu_k) = \hat{R}(\mu_1, \dots, \mu_k) + \lambda \cdot k,$$

where $\lambda \geq 0$ balances fit against complexity. Choose $\lambda$, then pick the $k$ that minimises this regularised loss. This is often easier to tune than picking $k$ directly, because many $\lambda$'s map to the same optimal $k$.

Elbow vs regularization — formally equivalent
Finding the elbow on the $\hat{R}$-vs-$k$ curve is exactly the same as finding the tangent of slope $-\lambda$ to that curve. The two heuristics are different visual framings of the same procedure.

7.3 Information-theoretic criteria

For probabilistic cluster models (e.g. Gaussian Mixture Models, which come next lecture), we can use classical model-selection criteria that balance likelihood against parameter count:

$$\operatorname{AIC}(k) = k - \log L, \qquad \operatorname{BIC}(k) = k \log n - \log L,$$

where $\log L$ is the maximised log-likelihood. Smaller is better. BIC penalises complexity more strongly than AIC because of the $\log n$ factor.

This is the global optimum for this (easy) dataset. But had we initialised with, say, $\mu_1^{(0)} = 1, \mu_2^{(0)} = 2$, Lloyd would still converge — possibly to a different, worse local optimum. That's exactly the motivation for k-means++.

Key Takeaways

  • Clustering is structured guessing without labels. It partitions data into groups so similar points stay together and dissimilar ones separate. It is one of three big unsupervised tasks, alongside dimensionality reduction and density estimation.
  • k-means = fixed cost, hard optimization. The objective $\hat{R}(\mu) = \sum_i \min_j \|x_i - \mu_j\|_2^2$ is clean and intuitive, but non-convex and NP-hard — so we rely on heuristics instead of exact solvers.
  • Lloyd's heuristic is coordinate descent. Alternating assignment and mean-update steps monotonically decreases the cost (Theorem 11.2), guaranteeing convergence to a local optimum in $\mathcal{O}(nkd)$ per iteration.
  • Initialization matters enormously. k-means++ combines randomness with squared-distance weighting to spread centers while resisting outliers, and comes with an $\mathcal{O}(\log k)$ expected-cost guarantee. It is now the default in scikit-learn.
  • Choosing $k$ has no textbook answer. Cross-validation fails because cost monotonically decreases with $k$. The elbow heuristic and the regularised objective $\hat{R}(\mu) + \lambda k$ are two views of the same workaround; information criteria (AIC, BIC) apply once we have a probabilistic model.
  • k-means bakes in strong assumptions. Using Euclidean distance forces spherical, comparable-size clusters. Elongated or curved shapes are misread — a limitation addressed by kernel k-means and (next lecture) Gaussian mixture models.
